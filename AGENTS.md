# AGENTS.md

## Cursor Cloud specific instructions

This repository is **Shizuku**, a native Android app (Kotlin/Java/C++ via Gradle). The
deliverable is an APK; there is no backend/web service or dev server, and no ports.

### Environment (already provisioned by the startup update script + VM snapshot)
- JDK 21 is the system default (`/usr/bin/java`), which is all Gradle needs — `JAVA_HOME`
  is optional.
- The Android SDK lives at `$HOME/android-sdk` (cmdline-tools, `platform-tools`,
  `platforms;android-36`, `build-tools;36.0.0`, `ndk;29.0.13113456`, `cmake;3.31.6`,
  plus `emulator` + `system-images;android-34;google_apis;x86_64`).
- `JAVA_HOME`, `ANDROID_HOME`/`ANDROID_SDK_ROOT`, and the SDK `PATH` entries are exported
  in `~/.bashrc`. Non-interactive shells (`bash -c ...`) do **not** source `~/.bashrc`, so
  for `adb`/`sdkmanager`/`emulator` either run a login shell or export
  `ANDROID_HOME=$HOME/android-sdk` and add `$ANDROID_HOME/platform-tools` /
  `$ANDROID_HOME/emulator` to `PATH` yourself. Gradle does **not** need these env vars
  because `local.properties` (`sdk.dir=$HOME/android-sdk`) is regenerated by the update
  script.

### Build / lint (see also `README.md` "Build" and `.github/workflows/app.yml`)
- Debug APK: `./gradlew :manager:assembleDebug` → `manager/build/outputs/apk/debug/*.apk`.
- Release APK requires signing (`signing.properties` + keystore); debug needs no signing
  (the `sign` signing config falls back to the debug keystore via `signing.gradle`).
- Lint: `./gradlew :manager:lintDebug`. Note the project currently has **pre-existing**
  lint errors (e.g. `MissingDefaultResource`); a failing `lintDebug` is the repo's existing
  state, not an environment problem.
- There are **no unit/instrumentation tests** in the repo (no `test`/`androidTest` source
  sets), so `testDebugUnitTest` has nothing to run.

### Non-obvious gotchas
- The `api` directory is a git submodule (`thedjchi/Shizuku-API`) and **must** be
  initialized or the Gradle build fails (`:aidl`, `:rish`, `:api`, etc. resolve under it).
  The update script runs `git submodule update --init --recursive`.
- `gradlew` may land without the executable bit; invoke as `sh ./gradlew ...` or rely on the
  update script's `chmod +x gradlew`.
- The repo was missing all binary image assets; the `mipmap-*` launcher/banner PNGs needed
  by `manager/src/main/AndroidManifest.xml` were restored (without them
  `:manager:processDebugResources` fails with `resource mipmap/banner not found`).

### Running the app (emulator)
- This VM has **no KVM** (`/dev/kvm` absent), so the emulator must run fully software
  emulated: `emulator -avd <name> -no-accel -gpu swiftshader_indirect -no-window
  -no-snapshot -no-audio`. Boot is slow (several minutes) and the emulated `system_server`
  frequently throws "Process system isn't responding" ANR dialogs — these are emulator
  performance artifacts, not app bugs; choose "Wait".
- Hello-world / core flow: `adb install -r <debug apk>`, launch
  `moe.shizuku.privileged.api/moe.shizuku.manager.MainActivity`, then start the privileged
  server exactly as the app's "connect to a computer" flow does:
  `adb shell <nativeLibraryDir>/libshizuku.so --apk=<installed base.apk path>`
  (find paths via `adb shell pm path moe.shizuku.privileged.api`). The home screen then
  shows "Shizuku is running".
